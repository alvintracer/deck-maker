# TranSight 발표자료 표준형 레퍼런스 가이드

## 0. 목적

본 문서는 `2026 국제 사기방지 컨퍼런스_v1.pptx`와 `BonanzaFactory_TranSight_Proposal_for_BANK_260520_순서_변경_3_2.pptx`의 시각적 흐름을 참고하여, 향후 TranSight·Bonanza Factory·Traverse 관련 발표자료를 일관된 “표준형” 스타일로 생성하기 위한 디자인 가이드이다.

본 가이드는 경찰청 제안 발표자료뿐 아니라 금융기관, 수사기관, 공공기관 대상 TranSight 제안서·세미나·IR·컨퍼런스 자료에 공통 적용한다.

## 1. 전체 디자인 원칙

### 핵심 키워드

- 신뢰감
- 공공기관 친화적
- 분석·수사 전문성
- 국내 기술 기반
- 정제된 B2B 제안서 톤
- 과장보다 근거 중심

### 표준형 스타일 정의

표준형은 화려한 스타트업 IR 스타일보다는, 공공기관·금융기관이 신뢰할 수 있는 **화이트 배경 + 네이비 헤더 + 정돈된 표/프로세스/다이어그램 중심** 스타일을 따른다.

### 지양할 것

- 과도한 그라데이션
- 지나치게 많은 아이콘
- 카드 UI 과다 사용
- 작은 글씨가 빽빽한 장표
- 경쟁사를 직접 비하하는 표현
- 법적·기술적으로 확정되지 않은 내용을 단정하는 표현

## 2. 슬라이드 규격

- 기본 비율: 16:9 Widescreen
- 기준 크기: 13.333 x 7.5 inch
- 여백:
  - 좌우: 0.55~0.70 inch
  - 상단: 0.35~0.45 inch
  - 하단: 0.30~0.40 inch
- 상단 헤더와 하단 footer를 유지하되, 제목 슬라이드와 섹션 브레이크는 예외 가능

## 3. 폰트 시스템

### 메인 폰트

- `나눔스퀘어_ac`
- 사용 위치:
  - 대제목
  - 섹션 제목
  - 표 헤더
  - 강조 문구
  - 핵심 숫자

### 서브 폰트

- `Pretendard`
- 사용 위치:
  - 본문
  - 설명 문장
  - 표 본문
  - 캡션
  - footer

### 폰트 크기 권장값

| 용도 | 권장 크기 | 비고 |
|---|---:|---|
| 표지 제목 | 34~40 pt | 짧고 강하게 |
| 섹션 제목 | 32~36 pt | 번호와 함께 사용 가능 |
| 일반 슬라이드 제목 | 26~32 pt | 한 줄 원칙 |
| 핵심 메시지 | 20~24 pt | 제목 아래 1문장 |
| 본문 | 15~18 pt | 5줄 이하 권장 |
| 표 헤더 | 13~16 pt | 네이비 배경 + 흰색 글자 |
| 표 본문 | 11~14 pt | 가독성 우선 |
| 캡션·주석 | 9~11 pt | 회색 처리 |
| Footer | 8~9 pt | 연한 회색 |

### 줄바꿈 규칙

- 한국어 단어 중간에서 줄바꿈되지 않도록 한다.
- 제목은 반드시 한 줄을 우선한다. 불가피하면 2줄까지만 허용한다.
- 영어 약어는 줄바꿈으로 분리하지 않는다. 예: `KYT`, `VASP`, `IAAN`, `TXID`, `AML`.

## 5. 레이아웃 시스템

### 기본 슬라이드 구조

1. 상단 소형 헤더
   - 좌측: 발표자료명 또는 섹션명
   - 우측: `보난자 컨소시엄` 또는 로고
   - 하단에 1px 네이비 라인

2. 본문 영역
   - 제목: 좌측 정렬
   - 핵심 메시지: 제목 아래 짧은 한 문장
   - 본문: 표, 프로세스, 다이어그램, 이미지 중심

3. 하단 footer
   - `CONFIDENTIAL — 무단 공개 금지`
   - 페이지 번호 선택
   - 연한 회색, 이탤릭 가능

### 표지 슬라이드

- 제목 중심, 여백 넓게.
- 하단에 날짜, 제안사, Confidential 문구.
- 로고는 과도하게 크지 않게 배치.
- 배경 이미지는 사용하지 않아도 됨. 사용 시 낮은 투명도 또는 우측 보조 이미지로 제한.

### 섹션 브레이크 슬라이드

- 큰 섹션 번호: `01.` `02.` 형식
- 섹션 제목: 32~36 pt
- 배경은 화이트 또는 아주 연한 블루.
- 하단 Confidential footer 유지.

## 6. 표 디자인

### 표준 표 스타일

- 헤더 배경: Deep Navy `#173A63`
- 헤더 글자: White `#FFFFFF`, 나눔스퀘어_ac Bold
- 좌측 카테고리 열: Pale Blue `#DDEBF7`
- 본문 배경: White 또는 `#F8FAFC`
- 선: `#C9D1D9`, 0.5~0.75 pt
- 표 안 텍스트는 중앙 정렬보다 좌측 정렬을 우선한다. 숫자는 중앙 또는 우측 정렬.

### 표 작성 원칙

- 한 슬라이드의 표는 5행 이하 권장.
- 6행 이상이 필요하면 두 장으로 분리한다.
- 문장형 설명보다 키워드 중심으로 쓴다.
- 표 안에서 굵은 글씨는 핵심 단어 1~2개만 사용한다.

## 7. 다이어그램 스타일

### 프로세스 다이어그램

- 가로 흐름: `접수 → 트리아지 → 분석 → 보고 → 후속지원`
- 단계 번호는 원형 또는 작은 배지로 표시.
- 각 단계는 1줄 제목 + 1줄 산출물만.
- 화살표는 얇은 네이비 또는 회색 사용.

### 파이프라인 다이어그램

- `데이터 수집 → 알고리즘 → 수사지원 → 서비스화 → 인프라화`
- 각 단계는 1개 박스.
- 배경은 화이트, 박스는 연한 블루/그린 계열.

### 비교 다이어그램

- 직접적인 경쟁사 비하 대신 “일반 분석지원 방식” vs “TranSight One-Team 방식”으로 표현 가능.
- 좌측은 회색, 우측은 네이비/블루 강조.
- 문구는 중립적으로 유지.

### 네트워크·트랜잭션 맵 이미지

- 실제 TranSight 트랜잭션 맵 이미지를 사용할 경우, 이미지 위에 과도한 텍스트를 얹지 않는다.
- 이미지 자체의 텍스트가 작으면 핵심 구간을 크롭하여 사용한다.
- 민감한 주소, 이름, 금액은 마스킹한다.

## 8. 이미지 사용 가이드

### 이미지 슬롯 표시 방식

이미지가 아직 없는 경우, 다음과 같은 프레임을 남긴다.

- 테두리: Dashed Line, `#9CA3AF`
- 배경: `#F3F4F6`
- 중앙 텍스트: `이미지 삽입 영역: TranSight Dashboard`
- 하단 캡션: 작은 회색 글씨

### 필수 이미지 유형

1. TranSight 대시보드 화면
2. 트랜잭션 맵 / 자금 흐름 그래프
3. IAAN 또는 해외 VASP 공조 구조
4. 보난자팩토리 로고
5. 트라버스 로고
6. 필요 시 경찰수사연수원 강의 또는 공공기관 협력 레퍼런스 이미지

### 민감정보 처리

- 지갑주소: 앞 6자리 + `...` + 뒤 4자리만 남김
- TXID: 앞 8자리 + `...` + 뒤 6자리만 남김
- 실명, 전화번호, 이메일, 사건번호는 마스킹
- 계좌·KYC·수사기관 내부 문서는 원본 이미지 사용 금지 또는 블러 처리

## 9. 슬라이드 유형별 템플릿

### A. 메시지 중심 슬라이드

- 제목 1줄
- 핵심 메시지 1문장
- 3개 근거 카드
- 우측 또는 하단에 작은 이미지 슬롯

사용 예:
- Executive Summary
- Why Bonanza Consortium
- 기대성과 종합

### B. 표 중심 슬라이드

- 제목 1줄
- 설명 1문장
- 3~5행 표
- 하단 주석 1줄

사용 예:
- 분석 항목별 대응범위
- 경찰청 지원 성과
- 기대성과

### C. 프로세스 중심 슬라이드

- 제목 1줄
- 5단계 가로 흐름
- 각 단계별 산출물 표기

사용 예:
- 분석 요청 처리 프로세스
- 수사 조치 연계 흐름

### D. 이미지 중심 슬라이드

- 제목 1줄
- 좌측 핵심 설명 3줄
- 우측 대형 이미지
- 이미지 하단 캡션

사용 예:
- TranSight 화면
- 트랜잭션 맵
- IAAN 구조

### E. 로드맵 슬라이드

- 제목 1줄
- 4~5단계 로드맵
- 단계별 색상은 블루→그린으로 자연스럽게 이동

사용 예:
- 단건 용역에서 국가 인프라화까지
- 고도화 계획

## 11. 경찰청 발표자료 전용 메시지 규칙

- “경찰청 수사관이 바로 활용할 수 있는 산출물”을 반복 강조한다.
- 법정·영장·거래소 협조 요청·환수 가능성이라는 실무 키워드를 활용한다.
- 국세청 환수 지원 사례는 핵심 레퍼런스로 쓰되, 다른 경찰청 지원 성과와 균형 있게 배치한다.
- 경찰수사연수원 강의 경험은 “분석 결과 설명력과 수사관 교육 역량”으로 연결한다.
- TranSight 라이선스 제공은 “무료 제공”보다 “사업기간 중 자체 분석·심층 검토 지원”으로 표현한다.

## 12. 은행·금융기관 발표자료 전용 메시지 규칙

- 규제 의무 강화, 실명계정 EDD, 온체인 AML, 스테이블코인, FATF R.15/R.16, KYC+KYT 통합을 강조한다.
- 은행 발표자료에서는 수사기관 사례를 보조 근거로 사용하되, 고객 책임·사전 경고·면책·내부통제 중심으로 재해석한다.
- API, Web, Batch Screening, Risk Code, Evidence, VASP Identification 등 기능을 명확히 보여준다.

## 13. 참고한 두 PPT의 반영 요소

### 2026 국제 사기방지 컨퍼런스 자료에서 반영할 요소

- 큰 섹션 번호와 간결한 Contents 구조
- 사례 기반 전개: 사건인지 → 일반화 → 조치 → 후속 모니터링
- 실제 범죄 흐름을 시간순으로 설명하는 타임라인
- 수사기관·거래소와의 조치 공조를 강조하는 구성
- 후속 모니터링과 풍선효과까지 이어지는 스토리텔링

### TranSight 은행 제안자료에서 반영할 요소

- `WHY NOW` 방식의 문제 제기
- 규제·범죄 리스크·스테이블코인이라는 거시 환경 정리
- 기업 개요·레퍼런스·기술력·데이터 수집 방법론의 신뢰감 있는 구성
- API/Web 기능을 분리해 설명하는 방식
- 실시간 모니터링, Multi-hop Analysis, 고도화 로드맵 표현

## 14. 표준 슬라이드 마스터 지시

Deck-Maker Agent는 다음 마스터 스타일을 기본 적용한다.

1. `Title Master`
   - 대제목 중앙 또는 좌측 상단
   - 부제 하단
   - 로고 영역 우하단
   - Confidential footer

2. `Section Master`
   - 섹션 번호 크게
   - 섹션 제목
   - 최소 장식

3. `Content Master`
   - 상단 헤더 라인
   - 좌측 제목
   - 본문 영역
   - footer

4. `Table Master`
   - 표 헤더 네이비
   - 좌측 구분열 라이트블루
   - 선 얇게

5. `Image Master`
   - 좌측 텍스트 / 우측 이미지 또는 상단 텍스트 / 하단 대형 이미지
   - 이미지 슬롯 프레임 지원

## 15. 최종 검수 체크리스트

- 한국어 단어 중간 줄바꿈이 없는가?
- 나눔스퀘어_ac / Pretendard가 적용되었는가?

- 경쟁사 비하 표현이 없는가?
- 기술적 한계가 있는 내용은 단정하지 않았는가?
