01
Executive Summary

핵심 제안: 기존 CODE 구현을 거의 그대로 활용

기술적 결론

빗썸 개발 공수는 거의 0에 가깝습니다.

TTR은 의도적으로 CODEVASP API 프로토콜을 그대로 사용합니다. 빗썸이 이미 CODEVASP에 연동되어 있다면, TTR 연결은 기술적으로 새 Travel Rule Hub 또는 새 VASP 라우팅 대상이 하나 추가되는 것과 유사합니다.

사업적 결론

빗썸은 CODE를 유지하면서 커버리지를 확장할 수 있습니다.

TTR은 CODE를 대체하는 구조가 아닙니다. 현재 CODEVASP 또는 VerifyVASP만으로는 금융기관 망분리·전용망·감사 환경을 직접 흡수하기 어려운 부분이 있으므로, TTR은 빗썸이 금융기관과 연결되는 추가 Trust Rail로 작동합니다.

05
Integration Scope

빗썸이 실제로 변경하면 되는 것

CODE 호환 모드 기준으로는 설정과 라우팅 분기만 추가하면 됩니다.

01

TTR Hub API URL 추가

환경변수 또는 설정 파일에 TTR endpoint를 추가합니다.

02

TTR Hub 공개키 등록

기존 Ed25519 키 관리 체계에 TTR 공개키를 1건 추가합니다.

03

라우팅 테이블 추가

TTR 대상 VASP 또는 금융기관 요청은 TTR Hub로 보내도록 분기합니다.

04

기존 테스트 재활용

헤더·암호화·payload가 동일하므로 기존 CODE 테스트 케이스를 재사용합니다.

const TR_HUBS = {
  code: 'https://trapi.codevasp.com',
  ttr:  'https://{TTR_ENDPOINT}/functions/v1'
};

const hub = isTtrTarget(vaspEntityId) ? TR_HUBS.ttr : TR_HUBS.code;

07
Value for Bithumb

빗썸이 TTR을 추가로 연결하면 얻는 것

KYT Atomic Gate

위험 주소로의 송금 시, PII를 수신 VASP에 전달하기 전에 자동 차단할 수 있습니다.

금융기관 접점

현재 CODEVASP 또는 VerifyVASP 중심 구조만으로는 금융기관의 망분리·전용망·보안·감사 환경을 충분히 흡수하기 어렵습니다. TTR은 보난자팩토리 Trust Rail을 통해 빗썸과 금융기관을 연결하는 별도 채널을 제공합니다.

해외 VASP 확장

GTR 경유로 Binance, Bybit, OKX 등 해외 주요 VASP와의 PII Verification 확장을 준비합니다.

스테이블코인 대비

스테이블코인 유통이 본격화되면 거래소는 레거시 금융기관과의 안정적인 규제 대응 채널이 필요합니다. TTR은 빗썸이 Finance on-chain 시대의 금융기관 접점을 선점하도록 지원합니다.

감사증적 통합

TranSight KYT 판정, Travel Rule 처리 결과, PII Verification 결과를 하나의 감사 로그로 통합하여 내부통제·사후심사·규제 대응에 활용할 수 있습니다.

기존 CODE 라인은 그대로 유지합니다. TTR은 국내 금융기관 및 글로벌 VASP 연결성을 보완하는 추가 rail입니다.

전략적 의미TTR은 CODE를 대체하지 않고, 빗썸의 금융기관 연결성을 보완하는 Trust Rail입니다. CODEVASP와 VerifyVASP는 기존 VASP 간 Travel Rule 처리에는 유효하지만, 향후 스테이블코인 유통을 담당할 금융기관의 망분리·보안·감사 요구사항까지 흡수하기에는 한계가 있습니다. TTR은 보난자팩토리의 Trust Rail과 TranSight KYT를 결합해, 빗썸이 국내 금융기관과의 디지털 자산 유통 채널을 선점할 수 있도록 지원합니다.

Phase 2 Option

TTR Direct Adapter는 향후 선택 사항입니다

초기 플러그 연결과 별개로, 빗썸이 TTR Network의 Domestic Direct Rail로 참여할 경우 추가 개발을 검토할 수 있습니다.

Phase 1

CODE 호환 모드

기존 CODE 구현 재사용. URL, 공개키, 라우팅 분기 중심. 개발 공수 1일 이내.

  • 빠른 연결
  • 낮은 리스크
  • 기존 운영 영향 최소
Phase 2

TTR-Bithumb Direct

금융기관의 TR 요청을 빗썸이 직접 수신·검증하는 구조로 확장.

  • 전용 API 수신 endpoint
  • 내부 KYC DB 기반 계정주 확인
  • 주소 소유 확인 API
제안 범위 본 제안의 1차 목적은 Phase 1의 저공수 연결입니다. Direct Adapter는 스테이블코인 유통량과 금융기관 온보딩 상황에 따라 단계적으로 추진합니다.
