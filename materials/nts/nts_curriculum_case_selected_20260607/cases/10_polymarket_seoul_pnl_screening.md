## Calculation Rule

```text
Total in = YES buy cost + NO buy cost
Total out = YES redeem amount + YES/NO sell proceeds
PnL = Total out - Total in
ROI = PnL / Total in
```

주의할 점:

- `redeem amount`만 보면 실제 손익을 오해할 수 있다. 같은 이벤트 안에서 `NO`를 샀다가 손절한 거래가 있으면 전체 손익은 달라진다.
- `Relay outflow amount`는 정산액보다 클 수 있다. 이 경우 기존 Polymarket 잔고나 다른 이벤트 수익이 섞였을 가능성이 있으므로, 서울시장 이벤트 수익으로 전액 귀속하면 안 된다.
- CEX 입금 확정은 Relay fill tx, 수신 주소 라벨, 후속 sweep까지 추가 확인해야 한다.

## Intermediate Applied Example: Large Withdrawal But Loss

중급 응용 교육에서는 row 3을 "큰 출금액이 있어도 해당 market 손익은 손실일 수 있다"는 케이스로 쓴다.

| Field | Value |
|---|---|
| Redeem row | `3` |
| Proxy wallet | `0x02982ce109fed9b564a493bb81cb34a5617e6606` |
| Seoul redeemed | `147,590.724528` |
| First external outflow | `172,655.128446` USDC.e |
| First outflow tx | `0x93f66bf0a9dbbe16a80a7e0b51abc5192a2e11b99babeba694743c5715d1aa8f` |
| First outflow destination | RelayDepository `0x4cd00e387622c35bddb9b4c962c136462338bc31` |
| Total in | `175,087.46` |
| Total out | `152,352.08` |
| PnL | `-22,735.38` |
| ROI | `-12.99%` |

해석:

- `172,655.128446` USDC.e가 외부로 나갔다는 사실은 출금 또는 bridge 후보 evidence다.
- 하지만 서울시장 market 손익은 `Total out - Total in`으로 봐야 하므로 `-22,735.38` 손실이다.
- 손실 원인은 `NO` 포지션 매수 `35,002.57`과 매도 회수 `4,761.36`이 섞인 점이 크다.
- 따라서 이 케이스의 교육 포인트는 `출금액 != 과세소득`, `정산액 != 순이익`, `market별 cost basis reconciliation 선행`이다.
