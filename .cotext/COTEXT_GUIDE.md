## 3. Block Format

Each entry ("block") in a cotext.md file:

```markdown
## YYYY-MM-DD HH:mm
<!-- source: me; author: YOUR_GITHUB_USERNAME -->

Your content here (markdown).

Attachments:
- ![image](./assets/filename.webp)
```

### Source Tags (Provenance)

| Tag | Who | Meaning |
|-----|-----|---------|
| `source: me` | Human | Human-authored block |
| `source: agent` | Any AI | AI-authored block |
| `source: claude` | Claude | Written with Claude |
| `source: chatgpt` | ChatGPT | Written with ChatGPT |
| `source: gemini` | Gemini | Written with Gemini |
| `source: antigravity` | Antigravity | Written with Antigravity |

**Rule: Never omit source and author metadata.**

## 6. Rules (All Agents)

1. **Always tag source and author** вҖ” Use `<!-- source: YOUR_NAME; author: YOUR_GITHUB_USERNAME -->` for every block
2. **Read before write** вҖ” Always read existing context before generating new content
3. **Append only** вҖ” Add new blocks at the end. Don't modify existing blocks
4. **Respect human content** вҖ” Blocks with `source: me` are primary. Don't summarize or rewrite
5. **Pull before push** вҖ” If using git directly, always pull latest first
6. **Markdown format** вҖ” All content must be valid markdown
